Mercury Options

This page allows you to configure mercury. Click the question mark in a section title for a description of how the individual options work.

Groups
?

Mercury allows you to open multiple tabs in one operation. Where a normal operation shows a list of the bookmarks that best match the input string, a group expansion searches for multiple input strings and can open the best match for each in separate tabs. For instance, when group expansion is enabled this:
so {do, re, mi}
will find the best matches for so do, so re and so mi and open the best match for each in a separate tab. You can use multiple groups at the same time, so
{o, so} {do, re}
will expand to o do, o re, so do and so re.
Enable group expansion
unchanged
If you use group expansion a lot you may sometimes find yourself often using the same groups. To make that easier mercury supports group variables which lets you define a variable that will be replaced with a group. For instance, if you often find yourself writing
sv {a, b, c, d, e}
you can simplify this by defining a bookmark, let's call it p, that points to the fake URL
set://a/b/c/d/e/
Now you can just write
sv $p
and you will get the same effect as if you'd written the full expression above.
Enable group variables
unchanged
Yes, the set://... syntax is weird but it's there to play nice with the bookmark manager.
